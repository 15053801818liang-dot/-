## 测试方法

1. **数据生成**: 随机生成 N 条规则 + 2N 条事实（谓词从 500 个候选池随机选取）
2. **插入**: 批量添加（跳过逐条一致性检查，`detect_recursive` 统一执行一次）
3. **一致性检查**: 首次完整运行 vs 二次缓存命中率对比
4. **查询延迟**: 10 次 `query_best_with_trace` 平均耗时（含变量查询和常量查询）
5. **对象计数**: 规则数、事实数、Term 参数槽数

## 优化方案

### 1. 谓词索引 (Predicate Index)
```python
predicate_index: Dict[str, List[Rule]]
```
规则按 head 谓词名建立索引。回溯时只遍历 `predicate_index[goal.name]`，
跳过所有不相关谓词的规则。

### 2. 事实索引 (Fact Index)
```python
fact_index: Dict[str, List[Term]]
```
事实按谓词名建立索引。回溯时只遍历 `fact_index[goal.name]`，
不遍历全部事实列表。

### 3. 一致性缓存
```python
_consistency_cache: Optional[HealthReport]
_cache_dirty: bool
```
首次一致性检查后缓存 `HealthReport`，KB 变更时设置脏标记。
后续检查直接返回缓存，O(1) 开销。

## 关键发现

| # | 发现 | 说明 |
|---|---|
| 1 | 谓词索引大幅加速查询 | 回溯时只遍历相关谓词的事实/规则，大幅减少无效 unify 调用 |
| 2 | 索引维护开销极小 | 插入时仅为 O(1) 字典追加，对总耗时影响可忽略 |
| 3 | 一致性缓存消除重复计算 | 对频繁执行检查的场景（如逐条 add_rule）提升显著 |
| 4 | 规模越大收益越高 | KB 中无关谓词越多，索引筛选效果越明显 |

## 结论

谓词索引和事实索引是知识库系统最基础也最有效的优化手段。
它们以极低的维护成本（O(1) 插入更新）换取显著的查询加速。
一致性缓存在需要频繁验证的场景下进一步消除冗余计算。
三种优化组合后，KB 系统在保持零外部依赖的同时，
查询性能可提升数倍。
